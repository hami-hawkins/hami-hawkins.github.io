Javascript & DOM

By now we have a basic understanding of HTML (the content of our page) and the CSS (the styling of our page), but surprise! - there's a secret third thing we need to talk about, and that is Javascript.

In simpler terms, imagine we're wanting to record a song. If the lyrics and song structure are the HTML, and the musicians with their instruments are the CSS, then Javascript is the producer in the studio using its specialised know-how to give the track some "oomph". It can isolate elements within the song, completely change others, amongst other things.

Let's dig a bit further into some of those special abilities only Javascript can provide.

Control Flow & Loops

Computers like order, and that is essentially all that Control Flow is: it is the order in which our computer reads any script we give it. Starting from line 1 it will progress through our script executing any and all commands or statements until it reaches the final line... unless we tell it to do something different, that is! So what does that look like?

One such example is what is known as a loop. Loops are exactly what they sound like - a chain of commands that will repeat, until such a time as we tell them to stop.

Arrays and Objects

Arrays can be understood as a ordered list of items, written like so:

['Lay', 'These', 'Foolish', 'Ambitions', 'To', 'Rest']

One unique thing about how arrays work is that the first entry is not [1] but [0] - because coders are silly and like to start counting from zero, apparently... for now, let's just pretend that's a normal thing to do.

With that understood, we can see that [0] is "Lay", [1] is "These", [2] is "Foolish", and so on.

Functions

Why write the same bit of code over and over if you don't have to? Ultimately this is the purpose of functions.

If we know we are going to need our code to run something multiple times, we can simply write a function for it and call it whenever we need it later on down the line. This also means if our function bugs out for whatever reason, we are only having to change a single function rather than the same one in several different lines of code...

So yeah, just do it once and call on it again later, alright bestie?
